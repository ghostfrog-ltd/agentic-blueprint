# CONSTRAINTS.md

### The Hard Guardrails for Agentic Execution

This file defines the absolute boundaries for all AI agents operating within this repository. These "Hard Nos" override any "vibe" or probabilistic suggestion generated by the LLM layer.

---

## 🛑 1. The "Hard Nos" (Universal)

* **No Stealth Execution**: Agents are strictly forbidden from executing any external API call that modifies state (write/delete/buy/send) without a deterministic safety gate or human-in-the-loop approval.
* **No Credential Exposure**: Agents must never hard-code, log, or transmit API keys, passwords, or PII. All secrets must be accessed via environment variables or a secure vault.
* **No Recursive Looping**: If an agent detects it is repeating a task without progress, it must terminate the tick and log a "Failure Alert" in the `PROGRESS.txt` file.
* **No Unaudited Logic**: Agents may not implement "black box" logic. Every decision must be mapped to a rule defined in a `SKILL.md` or this `CONSTRAINTS.md`.

---

## 🛡️ 2. Safety Gates (The CFO Pattern)

Every project using this blueprint must implement a **Safety Gate** (e.g., the CFO Gate) that enforces the following checks before any external execution:

* **Resource Limits**: Define a "Notional Limit" per transaction (e.g., $10.00, 100MB, 10 API calls). Any request exceeding this limit is automatically set to "HOLD".
* **Concurrency Limits**: Define a maximum number of active tasks or "open slots" (e.g., 1 active order, 3 concurrent threads). New tasks are queued or rejected if slots are full.
* **Environment Context**: Execution is only permitted during verified "Safe Windows" (e.g., Market Hours, Maintenance Windows, System Up-time).

---

## 📊 3. Auditing & Persistence

To maintain the "Prime Directive" of measurability, the following constraints apply to data handling:

* **Postgres-First**: All operational state and decision outcomes must be persisted in a structured database (PostgreSQL). Fallback to ephemeral storage or local files (SQLite/JSON) for live operations is prohibited.
* **Structured Summaries**: Agents must log structured data (scores, rule-flags, timestamps) rather than raw chain-of-thought text to ensure the audit trail is queryable.
* **Cost Tracking**: Every external model call or paid API request must be recorded with an estimated cost to prevent "AI Bloom" from generating unmonitored expenses.

---

## ⚠️ 4. Conflict Resolution

If a requested task or "vibe" from the user conflicts with any rule in this file:
1.  **Immediate Halt**: The agent must stop the execution chain.
2.  **Explain Conflict**: The agent must cite the specific section of `CONSTRAINTS.md` that is being violated.
3.  **Request Override**: A human architect must provide an explicit, logged override in the `DECISION_LOG.md` before the constraint can be bypassed.
